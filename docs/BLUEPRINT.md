# 八咫烏 公開基盤 設計図

## 固定方針

GitHubにすべて保存する
→ Streamlitで実験公開する
→ 価値を確認できたものを正式Webアプリ化する
→ 八咫烏ポータルから各アプリへ案内する

## 役割

- GitHub: コード、設計図、変更履歴
- Streamlit: すばやい試作と公開実験
- Cloudflare Pages: 正式Web版とポータル
- Supabase: 会員、個人設定、保存データ、通知

## 開発順

1. YATAGARASU VIEW
2. 八咫烏ポータル
3. VIEWの正式Web版
4. OCRなど次のアプリ

## 設計原則

- Guide, not decide.
- 結論を押しつけない
- 国内、海外、一次情報を分けて示す
- 事実、評価、推測を混ぜない
- 出典への道を必ず残す
- 海外報道も絶対視せず、複数情報源を比較する
- 利用者情報は任意回答とし、目的を明示する
- ニュース本文を無断転載しない
